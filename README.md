# SSH Arena — многопользовательская терминальная RTS

Многопользовательская терминальная RTS‑игра с элементами градостроительства и симуляции жизни. Игроки подключаются по SSH, видят мир в ASCII‑графике и управляют развитием своего поселения: строят здания, добывают ресурсы, заботятся о жителях, исследуют технологии, торгуют, воюют с соседями.

## Описание игры

**SSH Arena** — это стратегия в реальном времени, где каждый игрок управляет своим поселением в общем мире. Игра происходит полностью в терминале, используя ANSI‑цвета и символьную графику. Вы можете:

- **Строить** здания (жилые дома, фермы, шахты, казармы, исследовательские лаборатории).
- **Управлять жителями** — назначать им работу, следить за их потребностями (еда, отдых, безопасность).
- **Добывать ресурсы** (дерево, камень, железо, золото, еда) и развивать экономику.
- **Торговать** с другими игроками через рынок.
- **Исследовать технологии**, открывающие новые возможности.
- **Воевать** — создавать армии, атаковать соседей, защищать свою территорию.

Игра поддерживает одновременное подключение десятков игроков, синхронизацию состояния и сохранение прогресса.

## Требования

### Для игроков
- **SSH‑клиент** (OpenSSH, PuTTY, Termius, iTerm2, любое другое SSH‑совместимое приложение).
- **Терминал** с поддержкой ANSI‑цветов (рекомендуется true color) и Unicode.
- **Доступ к серверу** (IP‑адрес и порт, на котором работает игра).

### Для запуска сервера
- **Go 1.26+** (для сборки из исходников).
- **Docker** (опционально, для контейнерного развёртывания).
- **SQLite3** (встроен в сборку, отдельная установка не требуется).
- **SSH‑ключ** для сервера (генерируется автоматически при первом запуске).

## Быстрый старт

### 1. Получение кода

```bash
git clone <repository-url>
cd ssh-arena-app
```

### 2. Сборка из исходников

```bash
go mod download
go build -o ssh-arena ./cmd/server
```

### 3. Конфигурация

Отредактируйте файл `configs/config.yaml` под свои нужды. Основные параметры:

```yaml
server:
  host: "0.0.0.0"
  port: 8080           # порт для HTTP‑метрик
  max_players: 100
  tick_rate: 20        # частота игрового тика (мс)
  save_interval: 300   # интервал автосохранения (сек)

ssh:
  port: 2222           # порт для SSH‑подключений
  private_key_path: "./ssh_host_key"
  authorized_keys: "./authorized_keys"
  banner: "Welcome to SSH Arena!"
  idle_timeout: 300

game:
  world_width: 1000
  world_height: 1000
  chunk_size: 32
  starting_credits: 1000
  tax_rate: 0.05
  research_speed: 1.0
```

### 4. Запуск сервера

```bash
./ssh-arena
```

Либо напрямую через `go run`:

```bash
go run cmd/server/main.go
```

Сервер начнёт слушать SSH‑подключения на порту **2222** (по умолчанию) и HTTP‑метрики на порту **8080**.

### 5. Подключение игрока

```bash
ssh -p 2222 localhost
```

Если сервер работает на удалённой машине, замените `localhost` на её IP‑адрес или доменное имя.

После успешного подключения вы увидите ASCII‑интерфейс и сможете начать игру.

## Конфигурация

### Файл config.yaml

Полное описание всех секций:

- **server** — настройки сервера (хост, порт, лимиты).
- **game** — игровые параметры (размер мира, начальные ресурсы, налоги).
- **database** — настройки базы данных (драйвер, DSN, пул соединений).
- **logging** — уровень и формат логов.
- **ssh** — параметры SSH‑сервера (порт, пути к ключам, баннер).

### Где хранятся данные

- **База данных** — по умолчанию `./game.db` (SQLite).
- **SSH‑ключи**:
  - Приватный ключ сервера: `./ssh_host_key` (генерируется при первом запуске, если отсутствует).
  - Авторизованные ключи игроков: `./authorized_keys` (один ключ на строку, формат OpenSSH).
- **Логи** — выводятся в stdout (можно перенаправить в файл через systemd или Docker).

## Управление и команды

### Карта клавиш (внутри игры)

- **W/A/S/D** — перемещение камеры (вьюпорта).
- **E** — взаимодействие с объектом под курсором (построить, собрать ресурсы).
- **F** — атака (если под курсором вражеский юнит или здание).
- **/** — открыть текстовую командную строку.
- **1‑9** — быстрый выбор здания/юнита.
- **Space** — пауза/продолжение игры (только в одиночном режиме).

### Текстовые команды (через `/`)

- `/help` — показать список команд.
- `/build <тип> [x] [y]` — построить здание указанного типа.
- `/gather <ресурс>` — отправить жителей собирать ресурс.
- `/research <технология>` — начать исследование.
- `/attack <x> <y>` — атаковать указанные координаты.
- `/trade <игрок> <ресурс> <количество> <цена>` — предложить сделку.
- `/save` — принудительно сохранить игру.
- `/quit` — выйти из игры (сессия останется, можно переподключиться).

Полный список команд можно получить внутри игры через `/help`.

## Игровые механики

### Ресурсы

В игре пять основных ресурсов:

1. **Дерево** — добывается лесорубами из лесов.
2. **Камень** — добывается шахтёрами из каменных карьеров.
3. **Железо** — добывается из железных рудников, требуется для оружия и улучшений.
4. **Золото** — добывается из золотых жил, используется для торговли и найма специалистов.
5. **Еда** — производится на фермах, потребляется жителями.

### Здания

Каждое здание имеет свой функционал:

- **Хижина** — жильё для 5 жителей.
- **Ферма** — производит еду.
- **Лесопилка** — производит дерево.
- **Шахта** — производит камень/железо/золото (в зависимости от типа).
- **Казармы** — тренируют воинов.
- **Лаборатория** — ускоряет исследования.
- **Рынок** — позволяет торговать с другими игроками.

### Жители

Жители — основа вашего поселения. У каждого есть:

- **Профессия** (фермер, шахтёр, солдат, учёный).
- **Потребности** (еда, отдых, безопасность).
- **Уровень** — повышается со временем, увеличивает эффективность.

Управляйте жителями через меню или автоматизируйте с помощью политик.

### Бои

Боевая система пошаговая (в рамках игрового тика). Вы можете:

- Создавать юнитов (пехота, лучники, кавалерия).
- Атаковать вражеские здания и юнитов.
- Захватывать территории.
- Сооружать оборонительные стены и башни.

### Экономика

- **Налоги** — автоматически собираются с жителей (настраивается в конфиге).
- **Торговля** — игроки выставляют предложения на рынке.
- **Баланс** — следите за доходом/расходом, чтобы не обанкротиться.

### Исследования

Технологическое дерево включает ветки:

- **Сельское хозяйство** — улучшение ферм, новые культуры.
- **Промышленность** — более эффективная добыча, автоматизация.
- **Военное дело** — новые типы юнитов, укрепления.
- **Магия** (фэнтези‑режим) — заклинания, магические существа.

Исследования требуют времени и ресурсов, но дают постоянные бонусы.

## Docker

### Сборка образа

```bash
docker build -t ssh-arena .
```

### Запуск контейнера

```bash
docker run -d \
  --name ssh-arena \
  -p 2222:2222 \
  -p 8080:8080 \
  -v ./data:/app/data \
  -v ./configs:/app/configs \
  ssh-arena
```

### Docker Compose

Создайте `docker-compose.yml`:

```yaml
version: '3.8'
services:
  ssh-arena:
    build: .
    ports:
      - "2222:2222"
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./configs:/app/configs
    restart: unless-stopped
```

Запустите:

```bash
docker-compose up -d
```

### Монтирование томов

- **/app/data** — база данных и SSH‑ключи.
- **/app/configs** — конфигурационные файлы.

## Администрирование

### Управление сохранениями

Игра автоматически сохраняется каждые `save_interval` секунд. Ручное сохранение:

```bash
curl -X POST http://localhost:8080/admin/save
```

Файл базы данных `game.db` можно скопировать для бэкапа:

```bash
cp ./data/game.db ./backups/game-$(date +%Y%m%d-%H%M%S).db
```

### Мониторинг

Сервер предоставляет метрики в формате Prometheus на эндпоинте:

```
http://localhost:8080/metrics
```

Health‑check:

```
http://localhost:8080/health
```

### Логи

Логи выводятся в stdout в JSON‑формате. Для просмотра в реальном времени:

```bash
docker logs -f ssh-arena
```

Или при запуске без Docker:

```bash
./ssh-arena 2>&1 | jq .
```

### Обновление игры без потери прогресса

1. Остановите сервер.
2. Скопируйте `game.db` в безопасное место.
3. Обновите код (git pull) и пересоберите.
4. Запустите новую версию сервера с той же базой данных.

Миграции базы данных выполняются автоматически при изменении схемы.

## Устранение неполадок

### SSH‑подключение не работает

- **Проверьте, что сервер слушает порт 2222:** `sudo netstat -tlnp | grep 2222`
- **Убедитесь, что firewall разрешает подключение:** `sudo ufw allow 2222/tcp`
- **Проверьте логи сервера** на наличие ошибок аутентификации.

### Нет цветов в терминале

- Убедитесь, что ваш терминал поддерживает ANSI‑цвета. Проверьте переменную окружения `TERM`.
- Для PuTTY включите опцию «Allow terminal to use xterm 256‑color».
- Для Windows Terminal убедитесь, что включена поддержка цветов.

### Игра тормозит или лагает

- Увеличьте `tick_rate` в конфиге (например, до 50 мс).
- Уменьшите `max_players`, если игроков слишком много.
- Проверьте нагрузку на CPU и память.

### Не сохраняется прогресс

- Убедитесь, что у процесса есть права на запись в директорию с базой данных.
- Проверьте, что `save_interval` не равен 0.
- Посмотрите логи на наличие ошибок SQLite.

## Разработка

### Структура проекта

```
ssh-arena-app/
├── cmd/server/          # точка входа сервера
├── internal/            # основная логика
│   ├── engine/         # игровой цикл
│   ├── world/          # мир, чанки, тайлы
│   ├── building/       # здания
│   ├── citizen/        # жители
│   ├── combat/         # боевая система
│   ├── economy/        # экономика, торговля
│   ├── research/       # исследования
│   ├── events/         # шина событий, квесты
│   ├── network/        # SSH‑транспорт
│   ├── ui/             # ASCII‑рендеринг
│   └── storage/        # сохранения (SQLite)
├── pkg/                # вспомогательные пакеты
│   ├── config/         # загрузка конфигурации
│   ├── logger/         * структурированное логирование
│   └── errors/         * кастомные ошибки
├── configs/            # конфигурационные файлы
├── scripts/            # скрипты для сборки/развёртывания
└── api/                # protobuf‑спецификации (опционально)
```

### Добавление новой функциональности

1. Определите интерфейс в соответствующем пакете `internal/`.
2. Реализуйте интерфейс в инфраструктурном слое.
3. Внедрите зависимость через конструктор в `cmd/server/main.go`.
4. Напишите unit‑тесты (см. `*_test.go`).

### Логирование

Используйте глобальный логгер из `pkg/logger`:

```go
import "ssh-arena-app/pkg/logger"

log := logger.Get()
log.Info("message", "key", "value")
```

### Конфигурация

Все настройки загружаются через `pkg/config`. Добавьте новые поля в структуры `Config` и `config.yaml`.
